# 任务简介
## 待办清单项目

### 基础功能
1. 项目核心功能：
 一个基于原生js的待办清单应用,支持：
 - 添加、删除、编辑任务
 - 标记完成、未完成
 - 按全部、未完成、已完成筛选
 - 数据自动保存到1*Localstorage*，刷新不丢失

2. 添加任务怎么实现？空内容怎么处理
 - 通过按钮点击事件获取输入框的*value*
 - 使用if的条件语句,拦截空内容或纯空格输入
 - 当用户输入有效内容时,*todo*数组进行获取和渲染

3. 删除功能构想？
 当前版本尚未设置该功能，但是如果要加，可以设计一个回收数组，进行数据缓存，提供恢复入口

4. 完成状态为什么通过修改属性实现而不是直接改样式？
 - *completed*是数据层面的状态，样式是它的视图表现
 - 方便扩展：统计已完成数量，筛选，后端同步等
 - 遵循"数据驱动视图"原则

5. 数据存在哪里？为什么选Localstorage
 存在浏览器的localstorage中。
- 优点： API简单（getItem、setItem等简单指令即可）,无需后端,持久化储存
- 限制： 只能存字符串（JSON）、容量较小（仅5-10MB）

### 技术细节
1. 编辑功能遇到的困难？解决方案？
 - 预期想法：双击文本可以显示原文本，且输入框编辑完后恢复为文本
 - 解决：使用replaceChild把span换成input;监听blur，enter保存内容;使用e.stopPropagation防止冒泡事件

2. 为什么用replaceChild而不是innerHTML?
 - *innerHTML*会销毁原来的DOM节点，导致输入框失去焦点
 - *replaceChild*可以精准替换节点，保留焦点和事件

3. *filter*方法使用在哪里？会改变原数组吗？
使用场景：
 - 删除任务：*todos.filter(t => t !== todo)*
 - 筛选任务： *todos.filter(t => !t.completed)*
 - filter**不改变原数组**，返回一个新数组
